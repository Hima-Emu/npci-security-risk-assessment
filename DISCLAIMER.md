# Disclaimer

This repository documents a **self-driven academic project** and is published for educational and portfolio purposes only.

## Nature of the assessment

- This is a **qualitative, non-intrusive** risk assessment. It was conducted entirely using **publicly observable information** and industry-standard **passive reconnaissance** tools.
- **No login credentials were used.** No traffic was injected. **No vulnerabilities were exploited, probed, or tested.**
- All findings are based on data voluntarily broadcast by the target platform to any standard web browser or DNS resolver (HTTP response headers, DNS/TLS records, public cookie attributes, and publicly reachable paths like `robots.txt`).
- Internal systems, authenticated pages, APIs, and backend infrastructure were **explicitly out of scope** and were not accessed in any way.

## What this is not

- This is **not** a penetration test, red-team engagement, or formal security audit.
- This does **not** constitute professional security advice.
- The ISO 27001 / NIST CSF references in this repository are **observational mappings** used for structure and clarity — they are **not** compliance certifications or an official audit against those frameworks.
- The target organization referenced in this repository (NPCI) was not engaged, contacted, or involved in this project in any way. This is an independent, unaffiliated, unsolicited academic exercise.

## Responsible disclosure

No critical, exploitable vulnerability was identified during this assessment. Had one been found, it would not have been published here; responsible disclosure principles would have applied instead — i.e., reporting privately to the organization first, and only publishing after remediation or an agreed disclosure timeline.

If anything in this repository is believed to require a correction, clarification, or removal, please open an issue or reach out directly.
